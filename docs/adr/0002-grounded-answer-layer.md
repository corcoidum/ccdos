# ADR-0002: Grounded Answer Layer의 provider와 경계

**상태:** ADR-0003으로 대체됨
**날짜:** 2026-07-13

## 맥락

Phase 7의 retrieval-only 공개 위키 검색은 승인된 공개 기록만 검색해 출처와 발췌를 반환하며, 답변을 생성하지 않는다. 다음 단계인 Grounded Answer Layer는 검색된 근거를 바탕으로 자연어 답변을 생성하는 계층이다. 연결 전에 provider, 비용, 외부 전송 경계를 결정해야 한다는 것이 로드맵의 중단 조건이었다.

핵심 관찰: 이 시스템에서 답변 생성에 사용될 수 있는 데이터는 (1) 방문자의 질문과 (2) 이미 사람의 검토를 거쳐 공개된 S0_PUBLIC 콘텐츠뿐이다. 민감 데이터(CORCOIDUM-Core, ClinicOps-Local)는 저장소 밖에 있고 공개 index에 진입할 경로가 없다. 따라서 외부 전송 경계는 구조적으로 이미 충족되며, 실제 결정 대상은 품질·비용·남용 방어다.

## 검토한 선택지

| | 선택지 0: 유지 | 선택지 A: Claude API | 선택지 B: Cloudflare Workers AI |
|---|---|---|---|
| 내용 | retrieval-only 유지, 연결 보류 | 기존 Worker에 API 라우트 추가, Anthropic Claude 호출 | Cloudflare 엣지의 오픈소스 모델(Llama 등) |
| 한국어 품질 | — | 높음 | 제한적 |
| 비용 | 0 | 쿼리당 약 5–25원 (모델별, 아래 참조) | 무료 티어 있음 |
| 데이터 경로 | 없음 | 질문+공개 노트 → Anthropic | 질문+공개 노트 → Cloudflare (이미 호스팅 제공자) |
| 운영 부담 | 없음 | 낮음 (Worker secret + rate limit) | 낮음 |
| 세계관 부합 | 안전하지만 로드맵 정체 | 근거 인용·거부 규칙을 프롬프트로 강제 가능 | 인용 신뢰도 낮아 "추측하지 않는다" 약속과 충돌 위험 |

쿼리당 비용 추정 (질문+근거 약 2K 입력 토큰, 답변 약 300 출력 토큰 기준):
Haiku 4.5 약 $0.0035(≈5원) · Sonnet 5 약 $0.007(≈10원, 2026-08까지 인하가) · Opus 4.8 약 $0.0175(≈25원).
월 1,000회 조회 시 Haiku 기준 약 $3.5.

## 결정

선택지 A를 채택한다 (2026-07-13, corcoidum 승인).

- provider는 Anthropic Claude API, 시작 모델은 `claude-sonnet-5`로 한다. 모델은 비용·품질 관찰에 따라 설정 한 줄로 변경할 수 있다.
- 기존 정적 사이트를 서빙하는 Cloudflare Worker에 답변 API 라우트를 추가한다.
- 아래 경계 5항을 구현의 수용 조건으로 삼는다. 경계가 코드로 강제되기 전에는 8b를 배포하지 않는다.
- Phase 8a(클라이언트 검색 체험)를 먼저 공개하고, 8b(생성 연결)는 별도 작업으로 진행한다.

## 경계 (어느 선택지든 연결 시 적용)

1. **입력 경계** — 외부로 전송되는 것은 방문자 질문과 승인된 공개 index의 노트뿐이다. 다른 vault의 내용이 전송될 수 있는 코드 경로를 만들지 않는다.
2. **출력 경계** — 답변은 반드시 출처 인용을 동반하고, 일치하는 승인 근거가 없으면 생성하지 않고 "없다"고 답한다. retrieval-only의 약속을 생성 계층에서도 유지한다.
3. **비용 경계** — Worker 레벨 rate limit(IP당 제한)과 일일 호출 상한, provider 콘솔의 월 지출 한도를 함께 설정한다. 상한 도달 시 자동으로 retrieval-only로 폴백한다.
4. **키 경계** — API 키는 Worker secret으로만 보관하며 클라이언트 코드·저장소에 절대 포함하지 않는다.
5. **롤백** — 생성 계층 장애·비용 초과 시 검색 기능은 독립적으로 계속 동작한다.

## 단계적 도입

- **Phase 8a — 검색 체험 (LLM 없음):** 이미 번들된 공개 index를 클라이언트에서 검색하는 UI를 Lab에 공개한다. 외부 호출·비용 0으로 사용자 대면 검색 경험을 먼저 검증한다.
- **Phase 8b — 생성 연결:** 결정된 provider를 Worker 라우트로 연결하고 위 경계를 코드로 강제한다.

## 결과

장점: 데이터 경계가 구조적으로 안전한 상태에서 낮은 고정비로 로드맵의 마지막 단계를 진행할 수 있다. 단점: 공개 엔드포인트 노출로 남용 방어(경계 3)가 새로운 운영 책임이 된다.

## 구현 상태

Phase 8b의 provider 경계와 안전장치는 2026-07-14에 구현되었다. 같은 날 운영 provider와 credential은 사용자 결정에 따라 OpenAI Responses API로 변경했으며 자세한 이력은 ADR-0003을 따른다.
