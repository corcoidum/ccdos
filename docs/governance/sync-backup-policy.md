# 동기화와 백업 정책

## 데이터셋별 원칙

| 데이터셋 | 주 동기화 | 백업 | 복구 |
| --- | --- | --- | --- |
| ClinicOps-Local | 없음 (로컬 전용) | 암호화된 로컬/오프라인 사본 | 기기별 로컬 복구 |
| CORCOIDUM-Core | 승인된 개인 동기화 1개만 | 동기화와 독립된 버전 백업 | 버전 확인 후 수동 복원 |
| CORCOIDUM-Public | GitHub 원격 저장소 | 별도 내보내기 또는 릴리스 | Git 태그/릴리스에서 복구 |

## 운영 규칙

1. 동기화는 백업이 아니다. 같은 공급자·같은 계정의 복제본을 유일한 백업으로 사용하지 않는다.
2. 충돌이 발생하면 자동 병합 결과를 바로 믿지 않는다. 복사본을 보존하고 frontmatter, 링크, 발행 상태를 수동 대조한다.
3. 월 1회 합성 문서로 복원 연습을 기록한다.
4. 외부 동기화 앱에서 `ClinicOps-Local` 경로를 제외하고, 새 플러그인은 최소 권한·데이터 전송 여부를 검토한 뒤 allowlist에 추가한다.
