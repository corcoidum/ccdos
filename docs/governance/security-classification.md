# 보안 분류 정책

## 등급

| 등급 | 의미 | 저장 위치 | 외부 전송 |
| --- | --- | --- | --- |
| `S0_PUBLIC` | 공개해도 되는 승인 콘텐츠 | CORCOIDUM-Public, GitHub, 사이트 | 검증 후 허용 |
| `S1_PRIVATE` | 개인 학습·초안·비공개 맥락 | CORCOIDUM-Core | 명시적 검토 후 제한적 허용 |
| `S2_INTERNAL_DEIDENTIFIED` | 식별자를 제거한 운영 지식 | ClinicOps-Local | 기본 금지 |
| `S3_RESTRICTED` | 식별 가능 정보 또는 민감 원문 | 이 저장소에 저장하지 않음 | 절대 금지 |

## 강제 규칙

- 등급·visibility·publish_state가 하나라도 빠지면 **거부**한다.
- `S3_RESTRICTED`는 GitHub, Discord, 클라우드 저장소, 클라우드 LLM, RAG, 공개 사이트로 이동할 수 없다.
- `S2_INTERNAL_DEIDENTIFIED`도 기본적으로 외부 전송하지 않는다. 외부 공유가 필요하면 원본과 분리된 공개 후보를 새로 작성하고 검토한다.
- 실제 환자·직원 이름, 연락처, 주민등록번호, 차트번호, 진료 결과, 비밀값을 예시·테스트·로그에 넣지 않는다.
- 자동화는 분류를 추론하지 않는다. 명시적 메타데이터만 신뢰한다.

위반이 의심되면 작업을 멈추고 저장소에 쓰지 않은 채 수동 검토를 요청한다.
